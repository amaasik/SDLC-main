Initialization

Navigate to project folder

Command

$ git init

Initalizes a local Git repositiry

$ git status

Show the working tree status

Commits

Browse and inspect the evolution of project files

Command

$ git add .

Add file contents to the index

$ git commit -m "..."

Record changes to the repository

$ git log

Show commit logs

$ git checkout ... .

Switch branches or restore working tree files

$ git checkout -- .

Specifying -b causes a new branch to be created as if git-branch were called and then checked out.

$ git reset --hard commitIDwhereYouWantGoBack

Discards all history and changes back to the specified commit

Branches

Branches are an important part of working with Git. Any commits you make will be made on the branch you’re currently “checked out” to. Use git status to see which branch that is.

Command

$ git branch

Shows branch

$ git checkout ...

greats new branch named "new-feature"

$ git checkout -b new-feature

greats new branch named "new-feature"

$ git master

Change back to master branch

$ git merge ...

Join two or more development histories together

$ git branch -D new-feature

Deletes branch new-feature

GitHub

Synchronize your local repository with the remote repository on GitHub.com

Command

$ git fetch

Downloads all history from the remote tracking branches

$ git merge

Combines remote tracking branches into current local branch

$ git push

Uploads all local branch commits to GitHub

$ git pull

Updates your current local working branch with all new commits from the corresponding remote branch on GitHub. git pull is a combination of git fetch and git merge
